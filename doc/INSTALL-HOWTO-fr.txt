Installation

  Attention : 

    N'installez pas ce produit si vous n'avez pas besoin du SSO NTLM avec
    Active Directory

  Produits a installer :

    - LDAPUserGroupsFolder (>=2.5.0)

  Installation du produit :

    Il faut bien suivre la documentation présente dans LDAPUserGroupsFolder au
    sujet de l'installation de de la compatibilité Active Directory.
    Rien de spécial pour ce produit.

  Mise en place du SSO :

    Il faut commencer par remplacer l'objet cookie_authentification «Cookie Crumbler»
    à la racine de l'instance CMF, CPS ou Plone par un «NTLM Cookie Crumbler»
    cookie_authentification. Ensuite il faut vérifier que le patch NTLM pour
    l'acl_users courant est bien mis en place (ici c'est LDAPUserGroupsFolderPatch).

  Configuration apache2 :

    Il faut compiler le mod_ntlm pour apache2, donc avoir installé les
    bibliothèques de développement d'apache2. Les sources sont dans le
    répertoire «src» de ce produit.

  Hôte virtuel apache2 :

    Suivre exemple sur l'hôte virtuel présenté dans le fichier «vhost_sso.conf».
    Ne touchez pas aux occurences de «ntlm_remote_user». C'est sur ce paramètre
    que repose le SSO.
